Game requirements

Each game below is a todo: one GitHub issue per slug (label game). Ship when every Must item passes in Chrome.

Shared Must for all games:
- Runs in Chrome without install; static or step-based UI
- Puzzle data versioned under src/games/{slug}/data/ (CSV/JSON)
- Solo play; no network after first load
- Invalid moves show a clear message; optional hint and reveal-answer
- Keyboard accessible where clicks are primary input

sudoku
Must: 9x9 grid with 3x3 boxes; givens immutable; cell notes toggle; check row/column/box or full grid; reveal single cell or full solution with undo before lock
Should: difficulty tiers; import puzzle from 81-char string
Data: puzzles.csv — id, difficulty, givens, solution

chess
Must: board from FEN; side to move enforced; legal move generation; puzzle mode with stated goal; solution line validated
Should: arrow keys or click-drag; promotion picker; step backward through solution
Data: puzzles.csv — id, fen, goal, solution_moves (SAN or UCI)

checkers
Must: 8x8 draughts; forced capture; king promotion; multi-jump chains; win/draw detection
Should: hint for legal jump; puzzle goals
Data: puzzles.csv — id, position, side, goal, solution

go
Must: board sizes 9x9 and 19x19; place stone with suicide and ko rules; life-and-death puzzles; validate solution
Should: territory scoring tutorial; pass and reset
Data: puzzles.csv — id, size, setup, task, solution

logic-puzzles
Must: category grid with clues; deduction cells X/O; check consistency; single solution
Should: zebra/ordering templates; print-friendly layout
Data: puzzles.json — id, categories, clues, solution_grid

nonogram
Must: row/column run-length clues; fill/empty/unknown; solved picture matches bitmap
Should: sizes 5x5 through 25x25; binary fill v1
Data: puzzles.csv — id, width, height, row_clues, col_clues, solution

sorting-puzzle
Must: N tubes; pour top layer when rules allow; win when single color per tube; move counter and undo
Should: 4-8 tubes; preset level packs
Data: levels.json — id, tubes, moves_par

unscrew-puzzle
Must: layered bolts; only exposed removable; collision order; level complete when all removed
Should: undo; minimal animation; 20+ levels
Data: levels.json — id, layers, bolt_order_solution

bubble-shooter
Must: aim shooter; match 3+ on attach; gravity drop; step mode no timer; win/lose states
Should: ceiling drop every N shots; 3-5 colors per level
Data: levels.json — id, grid, colors, shots_limit

domino
Must: pip tiles chain from start; matching open ends; draw from boneyard; puzzle goal empty hand or complete chain
Should: no double spinner v1; drag to valid end only
Data: puzzles.csv — id, hand, chain_start, boneyard_order, solution_plays

spider-solitaire
Must: 10 columns; 1/2/4 suits; descending suited runs; deal row; win when foundations complete
Should: undo; highlight runs; one-suit mode
Data: optional deals.csv — id, seed, suits

n-queens
Must: N queens no mutual attack; toggle queen; attack lines on check; one/count/givens modes
Should: N 4-12; preplaced givens
Data: puzzles.csv — id, n, givens, solution or solution_count

crosswords
Must: blocked grid; across/down clues; check letter/word/full; reveal with confirmation
Should: symmetric grid optional; import from .puz or JSON v1
Data: puzzles.json — id, width, height, grid, clues, solution
